Project Overview

FoodSathi is a comprehensive food ordering and restaurant management system built with ASP.NET Core 6.0. The platform provides a complete solution for managing menus, processing orders, and handling user authentication with role-based access control.

Key Highlights

  • Modern Web Architecture: Built on ASP.NET Core 6.0 MVC pattern with Razor Pages
  • Secure Authentication: ASP.NET Core Identity framework with role-based authorization
  • Dual Database Design: Separated databases for authentication and business logic
  • Responsive Interface: Mobile-friendly design with modern UI/UX principles
  • Scalable Architecture: Entity Framework Core with code-first migrations
  • Production Ready: Built-in error handling, logging, and security features

Installation & Configuration Process

Project Extraction & Opening

Extract the project archive (ZIP file) to your preferred development directory. Avoid paths with spaces or special characters for best compatibility. Navigate to the extracted folder and locate the FoodSathi.sln solution file.

Open the solution in Visual Studio 2022 by:

  • Double-clicking the .sln file, or
  • Opening Visual Studio and selecting File → Open → Project/Solution
Recommended Path: C:\Projects\FoodSathi\ or similar short, simple path structure without spaces.

NuGet Package Restoration

Visual Studio will automatically detect and restore all required NuGet packages upon opening the solution. You can monitor the restoration progress in the Output window (View → Output).

If automatic restoration does not occur, manually trigger it:

  • Right-click the solution node in Solution Explorer
  • Select "Restore NuGet Packages"
  • Wait for the restoration process to complete (may take 2-5 minutes)
Expected Packages: Microsoft.EntityFrameworkCore (6.x), Microsoft.AspNetCore.Identity.EntityFrameworkCore (6.x), Microsoft.EntityFrameworkCore.SqlServer (6.x), Microsoft.EntityFrameworkCore.Tools (6.x), and related dependencies will be automatically restored.

Database Connection Verification

The application utilizes a dual-database architecture as configured in appsettings.json. Review the connection strings to ensure they match your environment:

"DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=FoodSathiDb;Trusted_Connection=True;MultipleActiveResultSets=true" "Menupage": "Server=(localdb)\\mssqllocaldb;Database=MenuDB;Trusted_Connection=True;MultipleActiveResultSets=true"

These connection strings are pre-configured for SQL Server LocalDB with Windows Authentication. No modifications are required for standard development environments. Both databases will be created automatically during the first application run.

LocalDB Instance: The default instance is (localdb)\mssqllocaldb. You can verify LocalDB installation by running sqllocaldb info in Command Prompt.

Entity Framework Migrations

Apply database migrations to create the FoodSathiDb schema. Open the Package Manager Console via Tools → NuGet Package Manager → Package Manager Console and execute:

Update-Database

This command will:

  • Create the FoodSathiDb database if it doesn't exist
  • Apply all pending migrations to set up the schema
  • Create Identity framework tables (Users, Roles, Claims, etc.)
  • Prepare the database for authentication and authorization
Migration Success: You should see output indicating successful migration application. If migrations don't exist, run Add-Migration InitialCreate first, then Update-Database.

Build the Solution

Compile the entire solution to ensure all dependencies are properly resolved and there are no compilation errors:

  • Select Build → Build Solution from the menu, or
  • Press Ctrl + Shift + B

Monitor the Output window for any build errors or warnings. A successful build should show "Build succeeded" with 0 errors.

Build Errors: If you encounter build errors, ensure all NuGet packages are properly restored and your .NET 6.0 SDK is correctly installed. Check the Error List window for detailed error messages.

Application Execution

Launch the application by pressing F5 (Start Debugging) or Ctrl + F5 (Start Without Debugging). The startup sequence will automatically:

  • Start the Kestrel web server
  • Create both database instances (FoodSathiDb and MenuDB) if they don't exist
  • Execute the role seeding routine to establish Admin and User roles
  • Provision the administrative user account with credentials shown above
  • Launch your default web browser to the application homepage
First Run: The first run may take 30-60 seconds as databases are created and seeded. Subsequent runs will be faster. The application typically runs on https://localhost:5001 or http://localhost:5000.

Database Architecture & Schema

The application implements a segregated database architecture for optimal separation of concerns, security, and scalability:

FoodSathiDb (Primary)
Purpose: Core application database for authentication and user management

Contains:
• AspNetUsers - User accounts and profiles
• AspNetRoles - System roles (Admin, User)
• AspNetUserRoles - User-role mappings
• AspNetUserClaims - User claims and permissions
• AspNetRoleClaims - Role-based claims
• AspNetUserLogins - External login providers
• AspNetUserTokens - Authentication tokens

Schema: Auto-generated via Entity Framework migrations
Connection String: DefaultConnection
MenuDB (Secondary)
Purpose: Business domain database for application-specific data

Contains:
• MenuItems - Food items, descriptions, pricing
• Categories - Menu categorization
• Orders - Order records and transaction history
• OrderDetails - Line items for each order
• Customer information and preferences

Schema: Created via EnsureCreated() on startup
Connection String: Menupage
Connection Management: Both databases utilize SQL Server LocalDB with Windows Authentication (Trusted_Connection=True). Connection pooling is enabled through the MultipleActiveResultSets parameter for optimized concurrent query execution and improved performance.

Database Relationships

  • One-to-Many: Users → Orders (one user can place multiple orders)
  • One-to-Many: Categories → MenuItems (one category contains multiple items)
  • One-to-Many: Orders → OrderDetails (one order contains multiple items)
  • Many-to-Many: Users ↔ Roles (through AspNetUserRoles join table)

Troubleshooting & Common Issues

Database Connection Issues

Problem: Cannot connect to SQL Server LocalDB

Solutions:
  • Verify SQL Server LocalDB is installed via SQL Server Object Explorer (View → SQL Server Object Explorer)
  • Check if LocalDB instance is running: open Command Prompt and run sqllocaldb info
  • Start LocalDB instance if stopped: sqllocaldb start mssqllocaldb
  • Reinstall "Data storage and processing" workload in Visual Studio Installer

Migration and Database Errors

Problem: Migration fails or database schema issues

Solutions:
  • Delete existing databases via SQL Server Object Explorer
  • Remove Migrations folder from project
  • Run: Add-Migration InitialCreate in Package Manager Console
  • Run: Update-Database
  • Rebuild solution and run again

Port Binding Conflicts

Problem: Application fails to start due to port already in use

Solutions:
  • Open Properties/launchSettings.json
  • Modify "applicationUrl" to use different ports (e.g., change 5001 to 5002)
  • Kill process using the port: Find PID with netstat -ano | findstr :5001, then taskkill /PID [number] /F

NuGet Package Restoration Issues

Problem: Packages fail to restore or build errors persist

Solutions:
  • Clear NuGet cache: Run dotnet nuget locals all --clear in Package Manager Console
  • Delete bin and obj folders from project directory
  • Close Visual Studio and delete .vs folder
  • Reopen solution and restore packages again
  • Check internet connection for package downloads

Static Files Not Loading

Problem: CSS, JavaScript, or images not displaying

Solutions:
  • Verify app.UseStaticFiles() is called in Program.cs
  • Check that files exist in wwwroot folder
  • Clear browser cache (Ctrl + Shift + Delete)
  • Check browser console (F12) for 404 errors
  • Verify file paths in view files are correct

Admin Account Not Created

Problem: Cannot login with admin credentials

Solutions:
  • Verify the SeedRolesAndAdminAsync method runs in Program.cs
  • Check Output window for seeding errors during startup
  • Delete FoodSathiDb database and run application again
  • Check AspNetUsers table in SQL Server Object Explorer for admin@foodsathi.com entry

Identity/Account Pages Not Found

Problem: Login/Register pages show 404 errors

Solutions:
  • Ensure ASP.NET Identity UI package is installed
  • Verify builder.Services.AddDefaultUI() is called in Program.cs
  • Check that app.MapRazorPages() is present in middleware pipeline
  • Scaffold Identity pages if they were customized

General Debugging Tips

  • Enable Detailed Errors: Set ASPNETCORE_ENVIRONMENT=Development in launchSettings.json
  • Check Logs: Review Output window and Debug console for error messages
  • Browser Console: Open F12 developer tools to check for JavaScript errors
  • Database Inspection: Use SQL Server Object Explorer to verify database structure and data
  • Breakpoints: Set breakpoints in controllers to debug request flow

Security Considerations

Development Environment Notice: This configuration is optimized for development, testing, and academic evaluation purposes. For production deployment, the following security enhancements MUST be implemented:

Required Production Security Enhancements

🔐
Password Policies
• Increase minimum password length to 8+ characters
• Require uppercase, lowercase, digits, and special characters
• Implement password complexity validation
• Enable password expiration and rotation policies
✉️
Email Confirmation
• Enable RequireConfirmedAccount in Identity options
• Configure SMTP settings for email delivery
• Implement email verification tokens
• Add resend confirmation email functionality
🔑
Two-Factor Authentication
• Enable 2FA for admin accounts (mandatory)
• Offer 2FA option for all user accounts
• Support authenticator apps (TOTP)
• Implement backup codes for recovery
🔒
HTTPS Enforcement
• Obtain and configure SSL/TLS certificates
• Enable HSTS (HTTP Strict Transport Security)
• Configure secure cookie policies
• Redirect all HTTP requests to HTTPS
🗝️
Credential Management
• Change default admin password immediately
• Use Azure Key Vault or environment variables
• Never commit credentials to source control
• Implement credential rotation policies
🔐
Connection String Security
• Store connection strings in secure configuration
• Use managed identities when possible
• Encrypt sensitive configuration sections
• Implement least-privilege database access
